## Discussion Summary

This was our first meeting after reading the project instructions. We started by going through the assignment requirements together to make sure we both understood what was expected. Since this project is different from the previous ones, we agreed that we should focus on the visualization aspect instead of spending too much time building complex models.

We decided to use the Titanic dataset because it already contains several interesting variables that can be visualized from different perspectives. It also fits the assignment well since survival was influenced by many different factors such as age, gender, passenger class, and fare.

One part of the assignment that immediately caught our attention was the requirement to intentionally create misleading visualizations. We both laughed because it felt strange to be told that misleading people was actually part of the assignment. Jay joked that this might be the only class where "lying with graphs" earns marks instead of losing them.

After discussing it further, we understood that the goal is not to deceive people, but to demonstrate how visualization choices can influence interpretation and why ethical visualization matters.

We also talked about how we wanted the notebook to look. Instead of making it look overly polished from the beginning, we agreed that it should follow a natural workflow where we first explore the dataset, understand the variables, clean only what is necessary, and then build the final visualizations.

### Decisions Made

- Use the Titanic dataset.
- Organize the notebook into logical sections.
- Create two accurate visualizations and two intentionally misleading visualizations.
- Keep the code readable and realistic rather than overly optimized.
- Store all work in the shared GitHub repository.
